# 《万界全面战争》AI部署策略策划文档

## 一、设计目标

### 1.1 核心问题
- **观赏性**：AI vs AI 对战时，双方应有战术博弈，而非固定套路
- **随机性**：每局部署应有变化，避免"每次开局都一样"
- **平衡性**：先手有优势，但不应"先手必赢"
- **战术深度**：AI应根据性格采取不同策略，而非千篇一律

### 1.2 设计理念
- **远程优先**：远程部队是核心输出，应优先部署
- **近战保护**：近战部队是盾牌，用来保护远程
- **抱团部署**：分散的部队容易被逐个击破，"抱团啃是最厉害的"
- **AI性格**：三种性格带来战术变化，增加随机性和观赏性

---

## 二、核心机制

### 2.1 有效距离（Effective Range）
**定义**：部署后有效距离 = 攻击距离 + 移动距离

**作用**：
- 选择先手第一只部队（A1）时，优先选择有效距离最高的远程部队
- B1放置时，确保不在A1的有效距离内

**计算方式**：
```javascript
effectiveRange = computeStats(ud).allowedRange + computeStats(ud).movement
```

### 2.2 放置规则补丁
**原规则**：敌方2格内不能放置部队

**新规则（优先级更高）**：**己方2格内可以放置部队**

**战术意义**：
- 允许己方部队紧密抱团（包括贴脸放置）
- 防止"敌方2格内不能放"导致无法保护远程部队
- 实现"骑兵贴己方弓兵"、"步兵贴敌方"的防御机制

## 三、AI性格系统

### 3.1 性格类型

| 性格 | 权重 | 战术风格 | 适用场景 |
|------|------|---------|---------|
| **进攻型（Offensive）** | 40% | 激进切入，试图快速突破 | 兵种质量高、有骑兵/飞兵 |
| **防守型（Defensive）** | 35% | 稳固防线，远程输出 | 兵种质量低、远程多 |
| **均衡型（Balanced）** | 25% | 攻守兼备，随机应变 | 兵种均衡 |

### 3.2 性格选择时机
- **选择时机**：每局对战开始时（掷骰后、部署前）
- **选择方式**：加权随机选择（进攻40% / 防守35% / 均衡25%）
- **存储位置**：`TurnState._aiPersonality[team]`

### 3.3 性格差异体现
性格差异体现在**位置选择**上（而非单位选择），确保双方都有远程对射能力：

| 性格 | 单位选择 | 位置偏好 |
|------|---------|---------|
| 进攻型 | 远程优先，后续优先骑兵/飞兵 | 靠近敌方，试图切入 |
| 防守型 | 远程优先，后续优先步兵/野兽 | 远离敌方，抱团构筑防线 |
| 均衡型 | 远程优先，按兵种优先级 | 抱团为主，远程尝试打到敌方 |

## 四、部署算法详解

### 4.1 先手方（A方）部署策略

#### 4.1.1 A1放置（第一只部队）
**选择单位**：
- 优先选择**远程部队**中有效距离最高的
- 如果没有远程部队，选择近战部队

**放置位置**：
- 放在**二环或三环**随机位置
- 原因：二环太靠前易被切入，四环太靠后远程打不到，二/三环最平衡

**示例**：
```
A1 = 远程部队（有效距离最高）
放置位置：二环或三环随机选择
```

#### 4.1.2 A2及后续部队
**单位选择**：
- 继续放**远程部队**，直到远程放完
- 远程放完后，按兵种优先级选择：
  - 近战 > 野兽 > 骑兵 > 飞兵

**放置位置**：
- **抱团**：选择能连接最多友军的位置（距友军≤1格）
- **远程输出**：尽量放在能打到敌方的位置

**特殊规则：进攻型第二只远程前置**
- 如果AI性格是**进攻型**且是第二只部队（`deployCount === 1`）：
  - 继续选远程部队
  - 位置选择：**靠近敌方**但**距A1不超过4格**
  - 战术意图：压迫敌方，但不远离主力

---

### 4.2 后手方（B方）部署策略

#### 4.2.1 B1放置（第一只部队）
**选择单位**：
- 所有性格都优先选择**远程部队**中有效距离最高的
- 原因：必须有远程对射能力，否则被先手方白嫖

**放置位置**：
- 放在**A1的中心对称方向**
  - 对称位置计算：`symHex = { q: -a1.q, r: -a1.r, s: -a1.s }`
- 确保**不在A1的有效距离内**
  - 即：`hDist(B1, A1) > A1的有效距离`
- 原因：避免被A1第一回合就攻击到

**示例**：
```
A1放置在 (2, -1, -1)
B1放置在 (-2, 1, 1)  [对称位置]
且 hDist(B1, A1) > A1的有效距离(假设为5)
→ B1不会被A1第一回合攻击到
```

#### 4.2.2 B2及后续部队
**单位选择**：
- 继续放**远程部队**，直到远程放完
- 远程放完后，按性格选择近战：
  - 进攻型 → 优先骑兵/飞兵（高机动切入）
  - 防守型 → 优先步兵/野兽（构筑防线）
  - 均衡型 → 按兵种优先级

**放置位置**：
- **抱团**：选择能连接最多友军的位置
- **根据性格调整**：
  - 进攻型：靠近A1（对方先手单位），试图切入
  - 防守型：远离敌方，抱团构筑防线，远程放后方
  - 均衡型：抱团为主，远程尝试打到敌方

---

### 4.3 防御机制

#### 4.3.1 触发条件
**检测时机**：每次放置前，检测是否需要触发防御机制

**触发条件**：
- 存在己方单位 `myUnit` 和敌方单位 `oppUnit`
- 且 `hDist(myUnit, oppUnit) === 3`（间隔2格）

**战术意义**：
- 敌方单位贴近己方（间隔2格），下一回合可能切入
- 需要立即防御：用骑兵保护远程，用步兵阻挡敌方

#### 4.3.2 防御响应

| 当前要放的兵种 | 防御动作 | 放置位置 |
|---------------|---------|---------|
| **骑兵/飞兵** | 贴己方弓兵（保护） | 在 `myUnit` 和 `oppUnit` 连线上，**靠近己方一侧** |
| **步兵/野兽** | 贴敌方部队（阻挡） | 在 `myUnit` 和 `oppUnit` 连线上，**靠近敌方一侧** |

**示例**：
```
敌方骑兵在 (3, -2, -1)
己方弓兵在 (1, 0, -1)
hDist = 3（间隔2格）→ 触发防御机制

如果我方放骑兵：
  → 放在 (2, -1, -1) 附近（靠近己方弓兵）

如果我方放步兵：
  → 放在 (2, -1, -1) 附近（靠近敌方骑兵）
```

## 五、兵种优先级

### 5.1 远程优先原则
**所有阵营**（先手/后手，所有性格）都遵循：
1. **第一只部队**：优先选择远程部队
2. **后续部队**：继续放远程，直到远程放完
3. **远程放完后**：按兵种优先级选择近战

### 5.2 兵种优先级排序
当远程部队放完后，按以下优先级选择：

| 优先级 | 兵种 | 原因 |
|-------|------|------|
| 1 | 步兵（infantry） | 高防御，适合前线 |
| 2 | 野兽（beast） | 高血量，适合坦克 |
| 3 | 骑兵（cavalry） | 高机动，适合侧翼 |
| 4 | 飞兵（flying） | 高机动+飞行，适合切入 |

### 5.3 性格差异（远程放完后）
| 性格 | 近战优先级 |
|------|-----------|
| 进攻型 | 骑兵/飞兵 > 步兵/野兽 |
| 防守型 | 步兵/野兽 > 骑兵/飞兵 |
| 均衡型 | 按标准优先级（步兵 > 野兽 > 骑兵 > 飞兵） |

### 5.4 兵种位置偏好
**高机动部队（骑兵/飞兵）**：
- ❌ 不要放在最前排（离敌方≤2格），会被集火
- ❌ 不要脱离大部队（离友军>3格）
- ✅ 放在侧翼（离敌方3-4格），保护侧翼，不突进太快

**低机动部队（步兵/野兽）**：
- ✅ 可以放在前排构筑防线
- ✅ 防守型AI：放在离敌方2-3格的位置（既不太近也不太远）

**远程部队**：
- ✅ 放在后排输出（已经在评分系统中处理）

## 六、位置选择算法

### 6.1 评分系统
`selectBestPosition` 使用评分系统选择最佳位置：

**基础分**：
- 抱团分：连接越多友军越好（`allyCount * 10`）
- 远程输出分：能打到越多敌方越好（`reachable * 5`）

**性格调整分**：
- 进攻型：靠近A1加分（`(10 - dToA1) * 2`）
- 防守型：远离敌方加分，远程更靠后加分
- 均衡型：远程打到敌方加分

### 6.2 特殊规则优先级
1. **A1放置**：二环或三环（最高优先级）
2. **B1放置**：对称位置（最高优先级）
3. **防御机制**：间隔2格触发（最高优先级）
4. **进攻型前置**：第二只远程靠近敌方（高优先级）
5. **通用抱团**：评分系统（默认）

## 七、战术示例

### 7.1 示例1：标准对局（均衡型 vs 均衡型）
```
先手方（A方，均衡型）：
  A1 = 弓兵（有效距离5），放在二环 (2, -1, -1)
  A2 = 弓兵（有效距离4），放在A1旁边 (2, -2, 0)
  A3 = 弓兵（有效距离3），放在A1/A2连线后方 (1, -1, 0)
  A4 = 步兵，放在弓兵前方 (3, -2, -1)

后手方（B方，均衡型）：
  B1 = 弓兵（有效距离4），放在对称位置 (-2, 1, 1)
  B2 = 弓兵（有效距离3），放在B1旁边 (-2, 2, 0)
  B3 = 步兵，放在弓兵前方 (-3, 2, 1)
  B4 = 弓兵（有效距离5），放在后方 (-1, 0, 1)
```

### 7.2 示例2：进攻型 vs 防守型
```
先手方（A方，进攻型）：
  A1 = 弓兵（有效距离5），放在三环 (3, -2, -1)
  A2 = 弓兵（有效距离4），放在靠近敌方一侧 (2, -2, 0)  [前置]
  A3 = 骑兵（有效距离6），放在能切入的位置 (1, 0, -1)
  A4 = 弓兵（有效距离3），放在骑兵后方 (2, -1, -1)

后手方（B方，防守型）：
  B1 = 弓兵（有效距离4），放在对称位置 (-3, 2, 1)
  B2 = 步兵，放在弓兵前方 (-2, 1, 1)
  B3 = 步兵，放在步兵旁边 (-2, 2, 0)
  B4 = 弓兵（有效距离5），放在后方 (-1, 0, 1)
```

### 7.3 示例3：防御机制触发
```
敌方骑兵放置在 (3, -2, -1)
己方弓兵在 (1, 0, -1)
hDist = 3 → 触发防御机制

我方下一回合：
  如果放骑兵 → 贴在 (2, -1, -1)（保护弓兵）
  如果放步兵 → 贴在 (2, -1, -1)（阻挡敌方骑兵）
```

## 八、效果预期

### 8.1 战术博弈
- ✅ 双方都有远程对射能力
- ✅ AI性格带来不同战术（进攻切入 / 防守防线 / 均衡对射）
- ✅ 防御机制让部署更动态（不是固定套路）

### 8.2 随机性
- ✅ AI性格随机选择（40% / 35% / 25%）
- ✅ A1/B1位置随机（二/三环随机、对称位置随机）
- ✅ 每局部署不同，增加观赏性

### 8.3 平衡性
- ✅ 先手有优势（远程优先部署），但不必赢
- ✅ 后手有对称放置+B1远程，可以抗衡
- ✅ 防御机制让劣势方有机会翻盘
