## 战斗系统总览

战斗系统基于《万界全面战争战场策划案》设计，采用回合制六边形棋盘对战。每场战斗分为 **掷骰 → 部署 → 战斗 → 结算** 四个阶段。

### 核心文件结构
| 文件 | 职责 |
|------|------|
| `combat.js` | 核心战斗引擎（伤害计算、士气、偷袭） |
| `turn-system.js` | 回合管理、状态机、记分板、地形 |
| `ai-engine.js` | AI决策系统（三档难度） |
| `hex-board.js` | 棋盘渲染、BFS寻路、动画系统 |
| `interactions.js` | 玩家交互（点击、移动、攻击） |
| `deployment.js` | 部署系统（独立模块） |

## 回合系统

### 回合状态机
```
dice（掷骰） → deploy（部署） → battle（战斗） → ended（结束）
```

### 战斗回合流程
```
回合开始（processTurnStart）
    ↓
包围状态检测（3+敌军相邻=被包围）
    ↓
双方交替行动（每方每个单位移动+攻击各一次）
    ↓
回合结束（processTurnEnd）
    ↓
士气恢复（未被攻击+未行动 +3）
    ↓
状态递减
```

### 后手方保护机制
- 第二玩家（后手方）第一回合获得 **30% 伤害减免**
- 标记：`TurnState.secondPlayer`

### 回合开始处理 (processTurnStart)
| 检测项 | 触发条件 | 效果 |
|--------|----------|------|
| 新被包围 | 3+敌军相邻 且 之前未被包围 | 士气 -5，添加"surrounded"状态 |
| 脱离包围 | <3敌军相邻 且 之前被包围 | 士气 +5，移除"surrounded"状态 |

### 回合结束处理 (processTurnEnd)
| 处理项 | 条件 | 效果 |
|--------|------|------|
| 士气恢复 | 本回合未被攻击 且 未行动 | 士气 +3 |
| 状态递减 | 所有状态 | 回合数 -1，过期移除 |

## 寻路与移动系统

### BFS寻路算法
统一的广度优先搜索（BFS）函数 `getReachableHexes()`，玩家和AI共用同一套寻路逻辑。

#### 阻挡规则
| 阻挡类型 | 地面单位 | 飞行单位 |
|----------|----------|----------|
| 山脉地形 | ❌ 不可通过 | ✅ 可通过 |
| 湖泊地形 | ❌ 不可通过 | ✅ 可通过 |
| 敌方单位 | ❌ 不可通过 | ❌ 不可通过 |
| 友方单位 | ⚠️ 可穿越但不可停留 | ⚠️ 可穿越但不可停留 |

#### 函数签名
```javascript
getReachableHexes(startHex, movement, isFlying, team)
// 返回: [{hex: {q,r,s}, dist: number}, ...]
```

#### 算法逻辑
1. 从起始格开始，BFS逐层扩展
2. 每格检查：地形阻挡 → 单位阻挡 → 决定是否可通过/可停留
3. 可通过但不可停留的格子（友军占据）继续扩展，但不计入结果
4. 最终返回所有可停留的格子及其距离

### 移动操作
#### 玩家移动
- 选中己方棋子 → 绿色高亮显示可达范围
- 点击目标格子执行移动
- 移动前触发 **偷袭检测**（`tryOpportunityAttack`）

#### AI移动
- 根据难度策略选择目标位置
- 移动距离受 `movement` 属性和难度 `maxMoveSteps` 双限制
- 移动后自动尝试攻击

#### 移动后的状态更新
| 状态 | 更新 |
|------|------|
| `_chargeDistance` | 记录本次移动格数（用于冲锋动能计算） |
| `_facing` | 朝向移动方向 |
| `_actionUsedThisTurn` | 标记本回合已移动 |
| `_didActionThisTurn` | 标记本回合行动过 |

## 核心战斗公式

战斗流程分为 **4个阶段**，完整模拟态势判定→战术计算→伤害结算→战损崩溃。

### 阶段0：数据准备
- 读取攻击方/防御方属性
- 脏数据消毒（NaN检测与修复）
- 弓兵近战封锁检查（弓类被近身禁止攻击）

### 阶段1：态势与士气预检定
#### 连锁溃逃
- 防御方2格内每有1支溃逃友军 → 士气 **-3**

### 阶段2：战术判定
#### 2.1 冲锋动能
仅骑兵/飞行单位近战攻击时触发：

| 移动距离 | 动能倍率 |
|----------|----------|
| 1格 | ×1.2 |
| 2格 | ×1.4 |
| 3格+ | ×1.6 |

#### 2.2 长柄武器抵御冲锋
防御方持长柄武器（`long`类型 + `E001`效果）：

| 情况 | 效果 |
|------|------|
| 贴脸未冲锋（距离≤1且冲锋距离≤1） | 仅抵消动能 |
| 有冲锋距离 | 抵消动能 + 反骑额外伤害（×1.5） |

#### 2.3 空军空中震慑
- 条件：攻击方为飞行单位 + 坐骑有 `E007` 效果
- 目标：步兵
- 效果：防御方士气 **-5**

#### 2.4 方位判定（正面/侧面/背面）
根据防御方朝向和攻击方位置计算：

| 方位 | 护甲降低 | 士气打击 | 骑兵冲锋额外伤害 |
|------|----------|----------|------------------|
| 正面 (front) | 0% | 0 | ×1.0 |
| 侧面 (flank) | 10% | -5 | ×1.2 |
| 背面 (rear) | 20% | -10 | ×1.4 |

### 阶段3：核心伤害公式

#### 基础伤害计算
```
基础伤害 = 武器基础伤害 - 有效护甲
         （最低为破甲值，保底1点）

有效护甲 = 总护甲 × (1 - 方位护甲降低比例)
```

#### 远程免伤
- 远程攻击时，盾牌提供 `rangedResist` 比例免伤

#### 弩手重甲克制
- 条件：弩类武器 + 目标护甲 ≥ 30
- 效果：伤害 **×1.2**

#### 最终伤害公式
```
最终伤害 = 基础伤害
         × 攻击范围（打多少规模）
         × 参与人数
         × 动能倍率
         × 骑兵方位加成
         × 弩手重甲克制
         × 后手减伤（第一回合0.7，仅防御方是后手时）
         + 长柄反骑额外伤害（如有）
```

#### 战损计算（血量→人数转换）
```
累计伤害 = 上次溢出伤害 + 本次伤害
阵亡人数 = floor(累计伤害 / 每单位血量)
溢出伤害 = 累计伤害 % 每单位血量

剩余人数 = max(0, 原人数 - 阵亡人数)
剩余血量 = 剩余人数 × 每单位血量 - 溢出伤害
```

### 阶段4：战损与崩溃
#### 战损士气惩罚
| 血量比例 | 士气惩罚 |
|----------|----------|
| ≤25% | -25 |
| ≤50% | -15 |
| ≤75% | -10 |

#### 崩溃判定
- 条件：士气 ≤ 0 且 基础士气 ≠ 100 且 非"永不溃逃"单位
- 效果：标记 `_routed = true`，从棋盘移除

#### 击杀士气奖励
| 击杀者血量 | 自身士气恢复 | 标签 |
|------------|-------------|------|
| <25% | +15 | 绝地反击 |
| <50% | +10 | 破釜沉舟 |
| ≥50% | +5 | （无标签） |

**友军鼓舞**：击杀者2格内友军士气 **+3**

## 偷袭系统

### 触发条件
| 条件 | 说明 |
|------|------|
| 移动方类型 | 非飞行单位 |
| 偷袭方类型 | 近战步兵（baseType === 'infantry'） |
| 距离 | 移动路径经过敌方步兵1格范围内 |
| 时机 | 移动方离开格子时触发检测 |

### 触发概率（按品阶）
| 品阶 | 概率 |
|------|------|
| 黑铁 (iron) | 20% |
| 青铜 (bronze) | 40% |
| 黄金 (gold) | 60% |
| 钻石 (diamond) | 80% |

### 偷袭效果
- **伤害**：偷袭者一次完整普攻伤害（全额，无衰减）
- **士气**：被偷袭方固定 **-5** 士气
- 多个潜在偷袭者时：随机选一个触发

### 参与规则
- 仅近战步兵可发动偷袭
- 空军永远不会被偷袭
- 每次移动最多被偷袭一次

## 士气系统

### 士气影响因素总览
| 因素 | 士气变化 | 触发时机 |
|------|---------|----------|
| 被包围（新触发） | -5 | 回合开始 |
| 脱离包围 | +5 | 回合开始 |
| 连锁溃逃（每支2格内溃逃友军） | -3 | 战斗开始 |
| 空军空中震慑（被空军攻击的步兵） | -5 | 战斗中 |
| 侧袭 | -5 | 战斗中 |
| 背袭 | -10 | 战斗中 |
| 血量≤75% | -8 | 战斗中 |
| 血量≤50% | -10 | 战斗中 |
| 血量≤25% | -12 | 战斗中 |
| 偷袭（被偷袭方） | -5 | 移动时 |
| 击杀敌人（自身） | +5~+15 | 击杀时 |
| 2格内友军击杀 | +3 | 击杀时 |
| 回合结束未被攻击且未行动 | +3 | 回合结束 |

> **叠加规则**：血量士气惩罚为叠加计算
> - ≤75%：仅-8
> - ≤50%：-8 + -10 = -18
> - ≤25%：-8 + -10 + -12 = -30

### 永不溃逃单位
- 标记：`_neverRout = true`
- 士气永远不会低于1
- 永远不会崩溃

## AI决策系统

### 三档难度对比
| 特性 | 简单 (easy) | 困难 (hard) | 传说 (legend) |
|------|-------------|-------------|---------------|
| 目标选择 | 最近敌人 | 血量低+远程优先 | 战术优先级（击溃>击杀>集火） |
| 移动策略 | 直线靠近 | 侧袭/背袭+抱团 | 分兵种战术+集火+风险评估 |
| 移动步数 | 由兵团属性决定 | 由兵团属性决定 | 由兵团属性决定 |
| 跳过回合概率 | 20% | 0% | 0% |
| 侧袭加成 | ❌ | ✅ | ✅ |
| 协同攻击 | ❌ | ✅ | ✅ |
| 阵型加成 | ❌ | ✅ | ✅ |
| 士气敏感 | ❌ | ❌ | ✅ |

### 难度配置可覆盖
- 代码默认配置 + `difficulty_config.json` 动态覆盖
- 字段：`targeting`, `moveStyle`, `aggression`, `preferRanged`, `flankingBonus`, `coordinatedAttack`, `formationBonus`, `skipChance`, `moraleSensitive`

> **注意**：`maxMoveSteps` 已移除，移动范围完全由兵团自身的 `movement` 属性决定，不再受难度限制

### 目标选择策略
#### 简单：最近目标 (aiTargetNearest)
- 直接选择距离最近的敌人

#### 困难：智能目标 (aiTargetSmart)
评分维度：
1. 血量低优先（<30%: +35, <50%: +20, <70%: +8）
2. 距离近优先（1格内: +18, 2格内: +10, 3格内: +4）
3. 远程单位优先（+22，需开启preferRanged）
4. 血量比例线性加分

#### 传说：战术目标 (aiTargetTactical)
优先级排序：
1. **能击溃**（HP击杀或士气击溃）：+500，可达再加+200
2. **3人集火**同一目标：+120
3. **2人集火**：+50
4. 士气危险线目标：+60（<30%）/ +40（<50%）/ +15（<70%）
5. 高血量伤害目标
6. 远程/高威胁单位：+25

### 移动策略
#### 简单：朝向移动 (aiMoveToward)
- 向目标直线移动，选距离最近的可达格

#### 困难：协同移动 (aiMoveCoordinated)
两种模式切换：
- **保守模式**（血量<40%）：优先抱团+远离敌人+能攻击就小加分
- **攻击模式**：走位质量（背袭>侧袭>正击）+ 靠近目标 + 抱团

兵种差异化：
- **弓兵**：保持2-4格距离，被贴脸就跑
- **骑兵/空军**：3格内加分（冲锋准备）

#### 传说：传奇移动 (aiMoveLegend)
分兵种精细化走位：

**骑兵/空军**：
- 背袭（+70）> 侧袭（+40）> 正击
- 优先切敌方远程（3格内+18，攻击范围内+30）
- 残血不减分（血再低也冲）

**远程单位**：
- 侧袭（+45）> 背袭（+30）> 正击
- 安全位加分（不会被反击 +20）
- 保持2-4格最优距离

**近战步兵**：
- 侧袭（+50）> 正击（+22）> 背袭（+15，背袭可能离战线太远）
- 抱团权重大（保护后排）
- 靠近己方远程单位（保护它们）

**通用：集火修正**
- 专注集火目标：攻击范围内 +12，距离越近越好

**传说独有：孤军深入风险评估**
```
敌方可攻击范围 = 敌方移动力 + 敌方攻击距离
友方可防御范围 = 友方移动力

若 目标位置在 3+ 敌方攻击范围内
   且 不在任何友方防御范围内
   → 分数 -120（高风险惩罚）
```

### 行动顺序排序 (aiSortActionOrder)
按优先级从高到低：

| 优先级 | 条件 | 加分 |
|--------|------|------|
| 1. 必逃 | 被贴脸的弓兵 | +1000 |
| 2. 必杀 | 能一击击杀敌人 | +800 |
| 3. 必攻 | 能攻击指挥官指定目标 | +600（可达）/ +300（移动后可达） |
| 4. 配合移动 | focus_ranged战术下的骑兵/空军 | +400 |
| 5. 推进 | 前线步兵（敌人在3格内） | +100 |
| 6. 站桩 | 远程原地输出 | 基础分 |

### 指挥官统筹层 (aiCommanderPlan)
回合开始时全局评估，决定战术方针：

| 战术 | 触发条件 | 说明 |
|------|----------|------|
| focus_ranged | 敌方有远程 + 我方有骑兵/空军 | 集火敌方最脆远程，骑兵切后排 |
| focus_weak | 敌方有残血单位（<30%HP） | 集火最弱敌人，扩大优势 |
| defensive | 我方总HP < 60% | 防守收缩，抱团保护 |
| aggressive | 默认 | 全面进攻 |

### 步兵偷袭应对（困难+传说）
- 近战非飞行单位已在敌方步兵1格内
- 移动+攻击范围内无可击杀目标
- → 不移动，原地交战（避免触发偷袭）

### AI主回合流程
```
1. 初始化AI状态（难度配置）
2. 计算阵型中心
3. 指挥官统筹评估 → 战术方针 + 行动顺序
4. 按行动顺序逐个单位处理：
   a. 能攻击到 → 选最优目标 → 尝试走位优化 → 攻击
   b. 不能攻击 → 选目标 → 移动 → 移动后尝试攻击
5. 全部处理完 → 结束回合
```

## 朝向与方位系统

### 朝向方向（0-5）
```
方向0: 右    (q+1, r)
方向1: 右上  (q+1, r-1)
方向2: 左上  (q,   r-1)
方向3: 左    (q-1, r)
方向4: 左下  (q-1, r+1)
方向5: 右下  (q,   r+1)
```

### 朝向规则
| 动作 | 朝向变化 |
|------|----------|
| 放置棋子 | 朝向棋盘中心 |
| 移动 | 朝向移动方向 |
| 攻击 | 朝向目标位置 |

### 方位判定 (getAttackAzimuth)
根据防御方朝向与攻击方位置，计算攻击来自哪个方位：

| 攻击方向与朝向的差值 | 方位 |
|---------------------|------|
| 0 (正前方) | front 正面 |
| 1 或 5 (两侧) | flank 侧面 |
| 2, 3, 4 (后方) | rear 背面 |

### 视觉表现
- 朝向三角形：在六边形边缘外侧，小三角指示方向
- 颜色：我方蓝绿色，敌方橙红色
- 设计原则：清晰可见但不遮挡棋子图标

## 武器类型与特殊规则

### 武器类型
| 类型 | 双手/单手 | 近战/远程 | 特殊规则 |
|------|-----------|-----------|----------|
| 剑 (sword) | 单手 | 近战 | - |
| 短剑 (shortsword) | 单手 | 近战 | - |
| 大剑 (greatsword) | 双手 | 近战 | - |
| 长柄 (long) | 双手 | 近战 | 抵御冲锋（E001） |
| 弓 (bow) | 双手 | 远程 | 被近身禁射 |
| 弩 (crossbow) | 低级单手/高级双手 | 远程 | 无视近战干扰，重甲克制 |

### 弓兵近战封锁
- 弓类武器：相邻格有敌军 → 完全无法攻击
- 弩类武器：不受近战干扰，被近身仍可射击
- 检查函数：`isBowBlockedByAdjacentEnemy()`

## 兵种分类与特性

### AI兵种分类 (aiClassify)
| 分类 | 判定 | 战术定位 |
|------|------|----------|
| bow | 弓类武器 | 惧怕近身，远程输出，保持距离 |
| crossbow | 弩类武器 | 不惧近身，远程输出，攻击优先 |
| cavalry | 骑兵baseType | 游走士气打击，侧袭背袭，切后排 |
| flying | 飞行baseType | 类似骑兵，优先切远程，可越地形 |
| beast | 野兽baseType | 类似步兵，正面冲锋 |
| infantry | 默认/步兵baseType | 抗线推进，抱团保护后排 |

### 野兽兵种限制
- 不可装备武器、盾牌、坐骑
- 仅可装备护甲（护甲命名格式："小型皮甲"等）

## 数据一致性保护

### NaN消毒
- 战斗开始时检查 `_currentHP`, `_currentCount`, `_accumulatedDamage`
- 若为NaN，用初始值覆盖
- 防止数值错误连锁扩散

### 安全计算
- 除法前除数检查（`Math.max(1, value)`）
- 数组越界保护
- 可选函数调用（`typeof func === 'function'`）

### AI回合安全守护
- 10秒超时强制结束回合
- 防止AI逻辑死锁导致游戏卡住
